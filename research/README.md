## Phase 2 着手時の調査計画

### 調査 05: Signal Protocol / MLS

**目的**: Phase 2 の「事前承認カプセル」を設計する際、エンドツーエンド暗号化された住所配信メカニズムが必要になる。Signal Protocol(X3DH + Double Ratchet)および MLS(Messaging Layer Security、RFC 9420)の住所配信への応用可能性を調査する。

**主な論点**:
- 一対多配信における鍵管理の複雑度
- Phase 2 ユースケース(EC 事業者への単発配信)に過剰な暗号機構ではないか
- より軽量な代替案(単純な公開鍵暗号 + ケイパビリティトークン)との比較

### 調査 06: Web Push 通知の標準化動向

**目的**: Phase 2 でユーザーに「事業者から住所要求が来た」ことを通知する仕組みが必要。Web Push API、VAPID、ブラウザ別の挙動、モバイルプッシュとの統合などを調査する。

**主な論点**:
- ゼロ知識アーキテクチャと通知ペイロードのプライバシー両立
- iOS Safari / Android Chrome / デスクトップブラウザ間の互換性
- セルフホスト時の通知配信戦略
