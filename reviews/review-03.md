#### [重大] ターゲティング否認とビジョンの矛盾

### Priya(i18n)

#### [重大] グローバル設計と Japan-only 法的立場の分裂

- libaddressinput 240 カ国対応 + Japan-only 規約は設計とマーケティングが食い違う

#### [中] カスタムハンドルの非 ASCII 文字対応

### Marcus(データ設計)

#### [重大] ネームスペース衝突

- ランダム形式 vs カスタムハンドルの 2 つのネームスペースが同じ空間
- ランダム形式は `XXXX-XXXX-XXXX` ハイフン区切り
- カスタムハンドルはドット必須で構文的に分離

#### [重大] ハンドル変更・解放時の参照問題

- ハンドルを解放すると、過去の参照先が別人になる
- 住所漏えい事故のリスク
- **推奨**: 一度取得したら解放しない(GitHub 方式)

#### [重大] アナリティクスとゼロ知識の両立設計

- プライバシー保全型(集計カウンタのみ)を提案
- Plausible/Umami をセルフホストで使う

## 臨時レビュー: 分散アーキテクチャ案

Taketo が提案: 「サーバを自分でホスティングできる、分散型という考え方はできないですか」

### 5 人レビュアーの反応

#### 解釈 1: 自己ホスト単独(Bitwarden 方式)

- **白石**: ★★★★☆ 強く支持
- **Priya**: ★★★★☆ 賛成
- **Marcus**: ★★★★☆ 既に計画済み
- **田村**: ★★★★☆ マーケティング価値あり
- **黒木**: ★★★★★ 現行計画と同じ

#### 解釈 2: 連合型(Mastodon/ActivityPub 方式)

- **白石**: ★★☆☆☆ プライバシー懸念多い(クロスインスタンス検索のメタデータリーク)
- **Priya**: ★★★☆☆ 条件付き賛成(地域特化が嬉しい)
- **Marcus**: ★★☆☆☆ 技術的複雑度激増
- **田村**: ★★☆☆☆ 認知拡大との両立が難しい
- **黒木**: ★☆☆☆☆ 強く反対(個人開発で扱える範囲を超える)

#### 解釈 3: 完全 P2P

- 全員却下(Phase 2 が原理的に成立しない)

### Taketo の重要な指摘

> 「将来の逆引き API を実現するには、各ホストを接続かつ、ルーティングみたいなのが必要になるってことですね」

これは本質的な洞察。**分散とPhase 2の両立にはルーティングが必要**。

### Claude の解答: メール型連合モデル

メール(SMTP)が同じ問題を解決している:

- 分散・自己ホスト可能・ルーティング可能・中央権威なし・連合
- 秘密: メールアドレスにルーティング情報が埋め込まれている(`[email protected]`)

GDA に適用すると:

- GDA コードに**インスタンス識別子を含む**: `taketo.yamaguchi@postkey.jp`
- DNS で routing(SRV レコード or well-known URI)
- 各インスタンスは完全自律
- インスタンス間の常時通信不要
- ActivityPub のような重い連合プロトコル不要

### 採用決定

**メール型連合モデル(handle@instance 形式)を採用**。

これにより:

- ビジョン vs 法的立場の矛盾が**構造的に解消**
- Phase 2 の技術的成立性が確保
- 自己ホストが「特典」から「アーキテクチャの核」に
- Mastodon 型連合の重さを回避

### 影響範囲

- GDA コード形式が `handle@instance.tld` に改訂
- API 仕様が「プロトコル仕様書」に格上げ
- プロジェクトの命名が 3 階層構造に
- 認知拡大戦略がプロトコルとサービスを区別する形に

## 主な決定(第 3 回総括)

1. スコープ現行維持(歯止め付き)
2. メール型連合モデル採用(DECISION-016)
3. 自己ホスト可能性を前面化(DECISION-017)
4. 1Password 流アナリティクス参考(DECISION-022)
5. 3 階層命名: OAP + Lodester + Postkey(DECISION-018)

## 残る課題

- DNS 発見メカニズムの詳細
- インスタンス識別子のルール
- インスタンス間の信頼レベル
- ユーザーのインスタンス間移行プロトコル
- プロトコルバージョン交渉
- オフラインモード時のクライアント挙動

これらはプロトコル仕様書の作成フェーズで解決される。
